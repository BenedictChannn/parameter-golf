# 2026-03-30 Agent Lab Harness Upgrade

## Objective

Implement the first practical upgrade pass on the autonomous research harness so the lab is better at:

- executing tranches consistently
- summarizing runs automatically
- reasoning from durable findings
- inspecting component budgets instead of guessing

## Work Completed

- added [`scripts/agent_lab/summarize_run.py`](../../scripts/agent_lab/summarize_run.py) to parse a finished run log into markdown, JSON, or short TSV-note summaries
- added [`scripts/agent_lab/analyze_budget.py`](../../scripts/agent_lab/analyze_budget.py) to break model params down by component family
- added [`scripts/agent_lab/run_tranche.py`](../../scripts/agent_lab/run_tranche.py) plus [`agent_lab/tranche_manifests/`](../../agent_lab/tranche_manifests/) for dry-run and sequential manifest-driven tranche execution
- added [`agent_lab/findings.md`](../../agent_lab/findings.md) as the durable conclusion surface
- added [`agent_lab/budget_report.md`](../../agent_lab/budget_report.md) for the current frontier shape
- upgraded [`scripts/agent_lab/plot_experiments.py`](../../scripts/agent_lab/plot_experiments.py) so the HTML dashboard now includes quick tables for top runs and recent keeps
- updated [`AGENTS.md`](../../AGENTS.md), [`agent_lab/program.md`](../../agent_lab/program.md), and [`agent_lab/README.md`](../../agent_lab/README.md) to describe the new workflow
- updated the repo and installed Codex `agent-lab` skills, plus the installed `engineering-worklog` skill, so the new workflow is part of the operating instructions

## Validation

Commands actually run:

```bash
python3 -m py_compile scripts/agent_lab/summarize_run.py scripts/agent_lab/analyze_budget.py scripts/agent_lab/run_tranche.py scripts/agent_lab/plot_experiments.py
python3 scripts/agent_lab/summarize_run.py --run-log agent_lab/run.log --exp-id AL-20260329-030 --anchor-exp-id AL-20260329-021 --hypothesis "smoke summary" --format md
python3 scripts/agent_lab/analyze_budget.py --num-layers 9 --model-dim 512 --num-heads 4 --num-kv-heads 2 --mlp-mult 2 --tie-embeddings 0 --artifact-total-bytes 15797032 --format md
python3 scripts/agent_lab/run_tranche.py agent_lab/tranche_manifests/template.json
python3 scripts/agent_lab/plot_experiments.py
```

Observed results:

- `summarize_run.py` extracted the expected `AL-20260329-030` metrics and rendered a reusable markdown block
- `analyze_budget.py` produced a stable component breakdown for the current best frontier shape
- `run_tranche.py` dry-ran the template manifest correctly and printed the resolved env for each candidate
- `plot_experiments.py` regenerated the dashboard successfully after the HTML enhancement

## Findings

- The harness was already good at remembering runs, but not yet good enough at remembering conclusions. `findings.md` fixes that gap directly.
- The main missing scientific aid was a budget view. The current frontier is clearly MLP-heavy, and the untied output head is small enough to justify further output-path experiments.
- The tranche runner is intentionally conservative in v1. It automates execution and local runtime logging, but still leaves tracked scientific closeout under human or agent judgment.
