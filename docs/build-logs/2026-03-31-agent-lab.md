## What Changed

The new default questions are:

- where is the model wasting parameters?
- where is it wasting compute?
- where is it wasting memory?
- where is it most fragile under quantization?
- which parts want dynamic routing instead of fixed routing?
- which parts need full global context, and which do not?
- which parts are helping training but not the compressed submitted model?

## Why This Matters

This gives the lab a better recurring lens for tranche design and post-run synthesis.

In practice, it should help in three ways:

- push tranche design toward architectural or mechanistic questions instead of shallow local tuning
- make post-run reflection more causal by asking what resource or routing problem the result exposed
- keep the submitted-model reality in view, especially compression and quantization, instead of optimizing only the fp training story

## AttnRes-lite Implementation

I implemented code support for the previously planned `T-20260331-Q` tranche.

New trainer surface:

- `ATTNRES_LAYERS`
- `ATTNRES_NUM_SOURCES`
- `ATTNRES_ROUTING_MODE=token_softmax|shared_scalar`

What the implementation does:

- adds a lightweight `AttnResidualLite` router inside selected blocks
- on selected later layers, replaces fixed skip-style reuse with a softmax mix over a few earlier depth states
- uses the current stream plus a small set of earlier sources rather than full all-layer depth attention

Candidate source order in the current implementation:

- current stream
- matched encoder skip for that decoder layer
- shallow encoder anchor
- root stream `x0` if more sources are requested

This keeps the mechanism lightweight and aligned with the tranche hypothesis: test dynamic depth routing without turning it into a full expensive all-layer attention-over-depth system.

Validation:

- `python3 -m py_compile agent_lab/train_gpt.py`
- a direct forward-pass smoke test with AttnRes-lite enabled on late layers succeeded
- `python3 scripts/agent_lab/run_tranche.py agent_lab/tranche_manifests/20260331-Q-attnres-lite.json` dry-ran successfully


## Tranche P Closeout

Tranche `P` answered the right question. The original hybrid winner was good, but it was not “done.”

The five runs told a clean story:

- [`AL-20260331-001`](../../agent_lab/experiments.tsv) lowered the mixer footprint from four lower layers to three and still improved the frontier to `1.3453`. That means the hybrid idea is robust; it was not just one lucky lower-four setting.
- [`AL-20260331-002`](../../agent_lab/experiments.tsv) pushed the idea harder with five lower mixers. That run got smaller and faster, but lost enough quality to fall behind the best lower-four line. So the answer is not “replace as much lower-stack attention as possible.”
- [`AL-20260331-003`](../../agent_lab/experiments.tsv) shrank the mixer width and stayed exactly flat to the older winner. That is useful because it says the winning placement is not terribly fragile to a modest mixer shrink, but it did not improve the frontier.
- [`AL-20260331-004`](../../agent_lab/experiments.tsv) widened the mixer and became the new best valid run at `1.3451`.
- [`AL-20260331-005`](../../agent_lab/experiments.tsv) widened the mixer kernel and also beat the older winner, though not the wider-mixer result.

Simple reading:

- lower-stack mixers are definitely real
- the family still has headroom
- the best current story is not “more mixer layers,” it is “a stronger mixer inside the same lower-four placement”

That matters. It means the frontier now believes the lower stack can stay mixer-heavy, but the mixer itself still needed more strength.

## Tranche R Closeout

Tranche `R` was mostly negative, but in a useful way.

The question was whether the stronger hybrid backbone wanted a different training tail near export. The answer is: mostly no.

- [`AL-20260331-011`](../../agent_lab/experiments.tsv) and [`AL-20260331-012`](../../agent_lab/experiments.tsv) both show broad warmdown still hurts.
- [`AL-20260331-013`](../../agent_lab/experiments.tsv) says a short stabilization phase is cleaner than a broad warmdown, but still not a win.
- [`AL-20260331-014`](../../agent_lab/experiments.tsv) is the only interesting near-miss. Head-focused cooldown nearly tied the older hybrid winner.
- [`AL-20260331-015`](../../agent_lab/experiments.tsv) shows the combined tail still does not stack into something useful.

So the new schedule lesson is:

- broad colder tails are still the wrong story
- output-path-sensitive cooldown is the only live hint
- schedule work remains secondary to architecture on this frontier

## Tranche Q Closeout

Tranche `Q` gave a strong answer, and it was mostly negative.

The core question was whether fixed skip and residual routing should give way to a small input-dependent router over earlier layer states. On this frontier, the answer is: not in the broad late-layer form we tested.

- [`AL-20260331-007`](../../agent_lab/experiments.tsv) tried the richer late-layer version with four source states and collapsed badly to `1.5043`.
- [`AL-20260331-009`](../../agent_lab/experiments.tsv) tested the cheaper shared-routing version and was just as bad at `1.5133`.
- [`AL-20260331-010`](../../agent_lab/experiments.tsv) stacked AttnRes-lite on top of the best cheap-routing package and still failed badly at `1.5016`.

So the key reading is that the failure is not a small implementation tax. Broad late-layer dynamic depth routing is simply the wrong routing shape for the current hybrid frontier.

There was one interesting exception:

- [`AL-20260331-008`](../../agent_lab/experiments.tsv) only applied AttnRes-lite to the top two layers and came in at `1.3499`.

That is still worse than the current best [`AL-20260331-004`](../../agent_lab/experiments.tsv) at `1.3451`, so it is not a win. But it is much cleaner than the other Q-family results. That suggests a more limited interpretation:

- if dynamic depth routing is useful at all here, it likely belongs only at the very top of the stack
- it does not look like a good broad replacement for the current fixed routing scheme

So tranche `Q` mostly closes the first AttnRes-lite branch. The family is not fully dead, but it is no longer a priority frontier path in its current form. The only version worth remembering is a much lighter top-of-stack-only revisit later, if stronger families flatten.
