## Plain-Language Technology Notes

I want the meaning of these tranches to stay readable later, so here is the short version of what each family was actually testing.

### What a Sequence Mixer Is

In the normal transformer block, every attention layer asks:
- which earlier tokens matter for this token?
- how much should I copy from them?

A sequence mixer is a cheaper alternative. It still mixes information across the sequence, but it does not build the full attention map. In this repo, the mixer acts more like:
- project the token stream into a smaller internal state
- run a lightweight learned sequence filter
- project back into the model dimension

Why try it?
- full attention is powerful, but expensive
- maybe early layers do not need full global token-to-token reasoning
- if a cheaper mixer is “good enough” in some layers, the model can spend budget more efficiently

### What a Low-Rank Output Head Is

The output head turns the final hidden state into logits over the vocabulary.

The dense untied head does this in one big matrix multiply.

A low-rank head says:
- maybe that big matrix is overbuilt
- instead of one large mapping, use two smaller mappings with a narrow middle

Why try it?
- fewer parameters
- maybe smaller artifact
- maybe the output mapping has hidden low-dimensional structure

### What Local-Global Attention Means

Normal attention lets each token look back at all earlier tokens.

Local attention limits that to a window, for example the last 256 tokens.

Why try it?
- maybe some layers only need nearby context
- maybe full global attention in every layer is wasteful
- if lower layers can stay local, later layers can still do the global cleanup

### What Skip / Residual Redesign Means

This model has several routing controls:
- `resid_mix`
- learned residual scales
- encoder/decoder-style skip links
- learned skip weights

The blunt question “can we delete them?” was mostly answered already: usually no.

The redesign question is different:
- can we keep the useful routing behavior
- but replace expensive or overly flexible parts with simpler gates?

That is why tranche `M` is more interesting than the older ablations.

### What Mechanism-Specific Learning Rates Mean

A normal training run gives broad parameter groups the same learning rate.

But not all parts of the model may want to learn at the same speed.

Examples:
- attention vs MLP
- early layers vs late layers
- newly introduced architecture pieces vs the old backbone

So tranche `N` was testing whether some ideas look weak only because they are being trained at the wrong relative speed.

### What Quantization-Aware Warmdown Means

The score is measured after the model is compressed and reloaded, not on the raw training weights.

So the late part of training might matter in a different way than usual:
- maybe a smoother ending makes the final compressed model behave better
- maybe the last updates make the weights more or less friendly to quantization

That was the point of tranche `O`.

It is not generic optimizer tuning.
It is trying to train for the model we actually submit.

## Tranche Reads

### J - Hybrid Sequence Mixer

- `J1` and `J2` both lost, so “replace the middle” was not the right generic story
- `J3` alternating attention and mixers worked and nearly beat the old frontier
- `J4` lower four mixer layers won hard and became the new best run
- `J5` upper four mixer layers stayed respectable, but was clearly weaker than lower-stack replacement

What I think:
- the family is real
- the placement matters a lot
- the early stack is where full attention was most overbuilt

Why I think this happened:
- early layers may mostly be doing rough representation building
- that job may not need full global token-to-token comparisons in every block
- later layers still keep attention, so the model has a place to do higher-fidelity routing and refinement
- the result fits a simple story: save expensive attention where it is least necessary first

### K - Output Head Architecture

- low-rank heads were bad
- bottleneck and two-stage nonlinear heads were also bad
- tied-plus-residual was the only version that looked at all plausible, but it was still clearly behind the dense untied head

What I think:
- tranche F was already telling the truth
- the win was the dense untied head plus calibration, not some hidden “better head architecture”
- this family is not where I would spend the next tranche on the current backbone

Why I think this happened:
- the dense untied head is probably not wasting capacity in the naive ways I tested
- low-rank heads likely throw away too much freedom
- richer nonlinear heads probably add the wrong kind of complexity for this tiny-model regime
- the good output-path gains came from giving the model direct output freedom, not from making the head more clever

### L - Local-Global Attention

- only `L1` ran
- it crashed before the first metric

What I think:
- this family is unresolved, not disproven
- I do not want to learn the wrong lesson from a crash
- it needs a real debug pass before any scientific judgment

Why this matters:
- a crash is not evidence against the idea
- if I blur “failed to run” into “performed badly,” I will accidentally kill a live direction
- this tranche needs debugging discipline before research judgment

### M - Skip And Residual Redesign

- shared scalar skip gates alone already improved over the old pre-program best
- sparse skip topology gave that gain back
- scalar `resid_mix` and shared residual scales stayed close but not best
- the full cheap-routing package stacked into `1.3534`, which is the second-best overnight result

What I think:
- tranche H was right that deletion was the wrong question
- the family wanted redesign, not removal
- this line is strong enough that it deserves a combo test with the hybrid-mixer winner

Why I think this happened:
- the model still wants routing help
- but it does not need every routing control to be full per-channel learned flexibility
- shared gates seem to give enough control at lower complexity
- that means the old residual family was not useless, just a bit over-parameterized

### N - Mechanism-Specific Learning Rates

- attention-up / MLP-down was the strongest subsystem split
- early-layer boost also looked slightly better than late-layer boost
- new-mechanism LR boost stayed near the old frontier
- none of them touched the new hybrid-mixer best

What I think:
- these are support tools
- they may matter more when paired with a promising new architecture than as a standalone frontier family

Why I think this happened:
- LR structure can help shape how the model learns
- but if the architecture itself is not stronger, LR alone usually only gives small nudges
- these results look like “good supporting technique,” not “new frontier family”

### O - Quantization-Aware Warmdown

- longer and earlier warmdown both hurt badly
- a short stabilization phase was only a near miss
- head-only cooldown almost tied the old best, but still lost
- the combined quant-tail also failed badly

What I think:
- the obvious “cool the model more before export” story is wrong
- if there is something real here, it is probably more output-path-specific and much subtler than the first pass

Why I think this happened:
- the frontier may still need strong learning signal late in training
- broad colder schedules may simply undertrain the model rather than make it more quantization-friendly
- the only remotely promising hint was output-head-specific cooldown, which matches the earlier finding that the output path is unusually important in this repo

## Durable Conclusions

- The overnight winner was architectural: hybrid mixers in the lower stack.
- The runner proved that bold block changes can beat the current transformer frontier on this stack.
- Skip/residual redesign is the best secondary family from the overnight program.
- Output-head architecture and naive quantization-aware warmdown are both de-prioritized for now.
- Local-global attention is still unfinished because the first candidate crashed.

## If I Were Explaining The Science In One Paragraph

The overnight program says the next gains are coming from changing how the model processes sequences, not from polishing the same transformer recipe. The big win was replacing early attention layers with cheaper sequence mixers. A secondary win came from simplifying the routing system without deleting it. In contrast, most attempts to invent a smarter output head or a colder quantization tail did not help. So the current research center of gravity should move toward better hybrid blocks and smart combinations of architecture wins, not back toward generic scalar tuning.
