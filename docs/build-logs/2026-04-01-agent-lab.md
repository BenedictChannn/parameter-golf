Meaning:

- not every block needs the same recipe

So the next phase is now defined much more cleanly: question the remaining sacred assumptions in the architecture, one by one, instead of defaulting back to familiar local tuning.

## Code Support For Y / Z / AB / AA

I then implemented the missing trainer support in [`train_gpt.py`](../../agent_lab/train_gpt.py) so those four tranches are actually runnable.

What changed, simply:

- `Y` support: new `MLP_STRUCTURE` variants let the model run no-expand quadratic token MLPs, no-expand linear-plus-quadratic token MLPs, and depth-selective lower-light / top-full MLP plans.
- `Z` support: `BLOCK_MODE=custom` plus `BLOCK_TYPES` lets different layers become `mixer_only`, `mixer_light`, `light`, `heavy`, or stay `standard`.
- `AB` support: `SHARE_BLOCK_GROUPS` reuses one canonical block across a group of layers and adds small per-layer scalar gains for the alias layers.
- `AA` support: `UPPER_ATTN_PLAN` now derives upper-stack attention-vs-mixer layouts from the hybrid frontier, and `ATTNRES_PLAN` adds the narrow top-only routing variant needed for the combo run.

The important engineering point for `AB` is honesty. Shared layers are deduplicated in `GPT.state_dict()`, and `load_state_dict()` fans the canonical weights back out to the alias slots. So the sharing tranche measures a real compressed-artifact effect, not a fake “shared at runtime but duplicated on disk” trick.

## Z1 Crash Investigation

When the live `YZABAA` program reached [`AL-20260401-051`](../../agent_lab/experiments.tsv), the failure turned out not to be in `AA` at all. It was the first `Z` run: the lower-two `mixer_only` block recipe.

The exact failure was a DDP unused-parameter error during warmup:

- `RuntimeError: Expected to have finished reduction in the prior iteration before starting a new one`
- unused parameter indices corresponded to the dead MLP-scale parameters on mixer-only blocks

The mechanism bug was simple:

- I removed the FFN path for `mixer_only` blocks
- but I still instantiated `mlp_scale` on those blocks
- so DDP saw trainable parameters that never participated in the loss

The fix was correspondingly narrow in [`train_gpt.py`](../../agent_lab/train_gpt.py):

- only create `mlp_scale` when `use_mlp_scale` is enabled **and** the block actually has an MLP

That preserves the block-uniformity experiment while removing the fake dead parameter path. The live `AB` process was already running on the old loaded code, so the fix applies to subsequent reruns of `Z` and anything after it, not retroactively to the already-started worker.

## Y / AB / AA Synthesis

With the completed results in hand, the picture is sharper than a simple “all lost” summary.

### Y: MLP Structure Minimalism

This tranche mostly closed the boldest version of the FFN-minimalism question.

- [`AL-20260401-046`](../../agent_lab/experiments.tsv) and [`AL-20260401-048`](../../agent_lab/experiments.tsv) were not gentle losses. They were clear proof that the current frontier is not just getting value from a pointwise polynomial token transform.
- [`AL-20260401-047`](../../agent_lab/experiments.tsv) says even halving the dense FFN width is too much simplification.
- [`AL-20260401-050`](../../agent_lab/experiments.tsv) says “full MLP only at the top” is also the wrong story.

The only run that kept any life in the family was [`AL-20260401-049`](../../agent_lab/experiments.tsv) at `1.3533`. That is still clearly worse than the frontier, but it is informative. It says:

- lower mixer-heavy layers may tolerate some FFN simplification
- upper attention-heavy layers still want the full dense FFN

So the deeper lesson is not “the FFN is unnecessary.” It is:

- the current frontier still needs real dense FFNs in most places
- but lower-stack FFN burden may still be slightly overbuilt

### AB: Compression-Native Sharing

This tranche did not produce a winner, but it taught something useful about where the model is *not* overparameterized.

- [`AL-20260401-061`](../../agent_lab/experiments.tsv) was the cleanest sharing result and still landed at `1.3511`
- [`AL-20260401-062`](../../agent_lab/experiments.tsv) and [`AL-20260401-063`](../../agent_lab/experiments.tsv) show that both lower-stage and top-stage whole-block sharing lose real function
- [`AL-20260401-064`](../../agent_lab/experiments.tsv) and [`AL-20260401-065`](../../agent_lab/experiments.tsv) show that reinvesting the saved bytes does not rescue the branch

That means the lower four mixers are not merely repeated copies of the same computation, and the top two attention layers are not trivially redundant either.

So the deeper takeaway is:

- naive whole-block sharing is too blunt
- the model may reuse *roles* across depth, but not enough to share full blocks cleanly
- the next compression-native branch needs a more selective structure than “tie these layers together”

### AA: Upper Attention Decomposition

This was the most scientifically interesting of the three completed families.

- [`AL-20260401-056`](../../agent_lab/experiments.tsv) says top-three-only global attention is somewhat survivable, but clearly worse
- [`AL-20260401-057`](../../agent_lab/experiments.tsv) says top-two-only global attention is too aggressive
- [`AL-20260401-058`](../../agent_lab/experiments.tsv) says top-only routing does not rescue an over-thinned upper stack
- [`AL-20260401-060`](../../agent_lab/experiments.tsv) says one final dominant reasoner layer is far too little

But [`AL-20260401-059`](../../agent_lab/experiments.tsv) at `1.3454` is different. It nearly tied the frontier while simplifying the upper stack by **interleaving** attention and lighter refinement blocks.

That is the most interesting architectural hint in this whole closeout.

The upper stack does not appear removable. But it may be reducible by **periodic global refresh** rather than contiguous full attention at every upper layer.

So the deeper lesson is:

- the upper stack still needs multiple global reasoning opportunities
- but they may not need to appear in every upper block
- interleaving is now a real live simplification idea

## Simple scientist read

The three-family read is:

- `Y` says the current frontier still needs real dense FFNs in most places
- `AB` says naive whole-block sharing is not the next compression-native breakthrough
- `AA` says the upper stack may be simplified only by interleaving, not by collapsing it

So the architecture theory is getting more precise:

- keep strong FFNs, especially in the upper half
- do not assume similar-looking layers can share full weights
- if simplifying the upper reasoning stack, think in terms of periodic global refresh, not top-only collapse

### AC: Conditional Heavy / Light Compute

This adds a `TOKEN_SELECTIVE_*` family of knobs.

The trainer can now:

- route a top-k subset of tokens into heavy FFN compute
- route a top-k subset into heavy attention while everyone receives a cheaper mixer-style light path
- combine those into one heavy-token path
- use either a learned router or a cheap norm proxy

That makes `AC` the first serious repo branch that asks whether the right unit of non-uniformity is the token, not only the layer.

### AD: Latent Upper Reasoner

This adds a `LATENT_REASONER_*` family of knobs.

The trainer can now:

- replace selected upper attention blocks with a compact latent reasoning module
- use either pooled latent summaries or learned latent-slot embeddings
- write a shifted latent context back into token space so the branch stays causality-safe

This is not a full Perceiver-style redesign, but it is a real test of the question “should the upper stack reason over a smaller latent workspace?”

### AE: Structured Sharing With Layer Deltas

This adds a `SHARE_DELTA_*` family of knobs.

The trainer can now:

- share attention, mixer, or MLP submodules across selected layer groups
- attach small low-rank per-layer output deltas so the layers are not forced to become identical
- dedupe those shared aliases from `state_dict` so artifact accounting stays honest

That makes `AE` a real second-generation compression-native branch instead of another blunt low-rank or whole-block-sharing repeat.

## Z / AC / AD / AE Closeout

The repaired `Z -> AC -> AD -> AE` queue mostly did what a good bold program should do: it killed one fake story cleanly, upgraded one vague clue into a real architectural signal, and revived one compression-native branch in a sharper form.

### Z: Block Uniformity Audit

This tranche ended up being much more useful than the initial crash made it look.

The wrong reading would be “the block template is fixed.” The right reading is narrower:

- [`AL-20260401-052`](../../agent_lab/experiments.tsv) says the lower mixer stage cannot simply drop its FFNs wholesale, even though doing so buys a lot of extra steps and a much smaller artifact.
- [`AL-20260401-054`](../../agent_lab/experiments.tsv) says broad lower-light blocks are also too weak.
- [`AL-20260401-053`](../../agent_lab/experiments.tsv) is the first real signal: alternating lower standard and mixer-only blocks stayed close.
- [`AL-20260401-055`](../../agent_lab/experiments.tsv) is the stronger signal: alternating heavy and light blocks across the stack nearly tied the frontier.

So the model may not want a uniform block recipe, but it also does not want “make the whole lower stage cheap.” The best current interpretation is:

- some expensive per-token refinement is still necessary
- but it may not need to happen at equal strength in every block
- periodic capacity concentration is now a live architectural clue

That is a much better answer than the initial vague question “does every block need both pieces?”

### AC: Conditional Heavy / Light Compute

This tranche was bold and mostly negative, but it gave a strong answer.

- [`AL-20260401-066`](../../agent_lab/experiments.tsv) was the least bad result and still lost clearly, even when selective compute was restricted to FFNs in only the top two layers.
- [`AL-20260401-067`](../../agent_lab/experiments.tsv) shows that spreading token-selective FFN routing across the whole upper stack is much worse.
- [`AL-20260401-068`](../../agent_lab/experiments.tsv), [`AL-20260401-069`](../../agent_lab/experiments.tsv), and [`AL-20260401-070`](../../agent_lab/experiments.tsv) all collapsed once global attention itself became token-selective.

The important part is not just “these runs lost.” The pattern is telling us something:

- the current frontier does not like sparse tokenwise global reasoning in this naive form
- the model seems to want dense, stable upper attention rather than a routed heavy-token attention path
- if token-selective compute ever works here, it will need to be much weaker and much safer than this first family

So `AC` mostly closes the naive heavy/light token-compute story. The only thing not fully dead is a very narrow top-only selective FFN idea, and even that is weak.

### AD: Latent Upper Reasoner

This branch is still not science.

[`AL-20260401-071`](../../agent_lab/experiments.tsv) crashed immediately on the first learned-slot latent-workspace candidate, before a metric was produced. So the correct research state is:

- not “latent upper reasoning failed”
- but “latent upper reasoning is unresolved because the first real candidate crashed”

That matters, because this branch was one of the boldest architecture questions in the queue. It needs debugging before it deserves any scientific judgment.

### AE: Structured Sharing With Layer Deltas

This was the most interesting positive surprise in the queue.

The first compression-native sharing branch (`AB`) mostly killed whole-block sharing. `AE` asked a sharper question: what if the structure is shared, but each layer gets a small learned correction back?

The results say that this was the right refinement of the question.

- [`AL-20260401-077`](../../agent_lab/experiments.tsv) was the strongest run: pairwise lower-mixer sharing with small deltas stayed close to the frontier.
- [`AL-20260401-080`](../../agent_lab/experiments.tsv) stayed similarly close while widening the mixer, which means the “share then reallocate” story is now real enough to keep alive.
- [`AL-20260401-076`](../../agent_lab/experiments.tsv) and [`AL-20260401-079`](../../agent_lab/experiments.tsv) show that upper-attention sharing is survivable but weaker than lower-mixer sharing.
- [`AL-20260401-078`](../../agent_lab/experiments.tsv) says FFN sharing is still the weakest target.

The deeper lesson is:

- the model may have some reusable lower-stage structure
- but it does not want blunt equality across layers
- preserving layer identity with small deltas matters

That is qualitatively different from both naive factorization and naive whole-block sharing. So `AE` keeps compression-native architecture alive, but in a much narrower and more believable form.

## What I Think Now

The new cross-tranche picture is:

- block uniformity is not sacred, but the good non-uniformity is periodic, not broad collapse
- token-selective compute was the wrong kind of non-uniformity
- compression-native sharing is still alive, but only with local sharing and preserved layer identity

So the model keeps repeating the same lesson:

**it rewards carefully placed asymmetry, not blunt asymmetry**

That matches almost the entire frontier story so far:

- lower mixers yes, all-over simplification no
- simple skip gates yes, broad routing no
- interleaved upper attention maybe, upper collapse no
- lower-stage structured sharing maybe, whole-block sharing no
