### E2 — GQA: `NUM_KV_HEADS=2` (env-only)

- **Hypothesis:** Halve KV heads → less KV-cache traffic and attention work → **more train steps in 600s** or lower loss at same step budget → better `final_int8_ttt_lora`.
- **Change:** `NUM_KV_HEADS=2` (still 8 query heads; GQA grouping 4:1).
- **Metrics:** `step:356` @ 600s; mid `val_bpb` **1.5865**; **`final_int8_ttt_lora` 1.5921**; peak **11711 MiB**.
- **Reflection:** vs baseline **1.6099** — **lower is better**; also **+14 steps** in same wall (356 vs 342).
- **Outcome:** **`keep`**. Landed default `NUM_KV_HEADS=2` in `Hyperparameters` → commit **`d9ca661`**.

---

### E3 — Half global batch (`TRAIN_BATCH_TOKENS=262144`)

- **Hypothesis:** Halving tokens/step cuts per-step work enough that **695 optimizer steps** fit in 600s vs ~356 — more updates → better compression even if batch noise is higher.
- **Change:** env `TRAIN_BATCH_TOKENS=262144` on top of E2 defaults.
- **Metrics:** `step:695` @ 600s; mid `val_bpb` **1.4477**; **`final_int8_ttt_lora` 1.4239**; peak **6114 MiB** (~6.0 GB).
- **Reflection:** Large win vs **1.5921**; VRAM headroom opens room for wider layers or longer eval later.
- **Outcome:** **`keep`**. Default `TRAIN_BATCH_TOKENS=262144` in code → commit **`262d70f`**.

---

### E4 — Higher `MATRIX_LR`

- **Hypothesis:** Muon matrix updates may be under-moved at 0.04; **0.06** improves loss trajectory in fixed wall.
- **Change:** env `MATRIX_LR=0.06` on E3 stack.
- **Metrics:** `step:696` @ 600s; mid `val_bpb` **1.4423**; **`final_int8_ttt_lora` 1.4129**; peak **5999 MiB**.
- **Reflection:** vs E3 **1.4239** — **improved**; landed default `MATRIX_LR=0.06` → commit **`bab9bf7`**.
